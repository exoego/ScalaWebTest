Contributing

Features, bugs and other issues

ScalaWebTest welcomes contribution of any form at any time. If want to report a bug, request a feature or change, just create an issue - a responsible committer will get to it.

Contributing source code

If you would like to contribute code, we're looking forward to your pull request. Note: The primary development branch is "master", so you might want to develop against that.

Before we can integrate your change, we need you to sign the contributor license agreement. This will be enforce by CLA assistant and is crucial to keep our stuff IP-clean - thank you!

Becoming a committer

If you have knowledge of Scala, ScalaTest and Selenium and would like to join, you are more than welcome as a committer. Usually, you start of by making a couple of pull requests to show your expertise - then, the existing committers will vote on the decision.
